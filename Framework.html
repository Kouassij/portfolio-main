Cahier des charges

Afin d'améliorer la sécurisation des accès aux applications métiers d'une entreprise par l'implémentation d'une solution d'authentification unique (Single Sign-On, SSO) via Keycloak, un système de gestion des identités et des accès (Identity and Access Management, IAM). Jusqu'à présent, l'authentification était gérée de manière indépendante pour chaque application, avec des identifiants (login/mot de passe) stockés dans leurs bases de données respectives. L'objectif est de centraliser l'authentification pour renforcer la sécurité et simplifier la gestion des accès.


Nos missions

La tâche spécifique assignée consiste à modifier une application existante développée avec le framework Symfony, qui actuellement utilise une authentification locale (identifiants enregistrés dans une base de données). Il est demandé de récupérer une version antérieure de cette application (sans l'authentification gérée par Symfony), de configurer Keycloak pour l'authentification SSO, et de faire évoluer le code de l'application pour intégrer cette nouvelle méthode d'authentification.
